Summoners War Statistics - Introduction

On this website you can check things like fastest dungeon comp, toughest monster, the most efficient rune or even different builds for certain units. Data is stored anonymously and everyone can contribute. There is no hard-coded data (like builds for monster on wikia), everything is from people who decided to contribute. If you want to be part of this project, check Contribute page (heart icon on the menu).

Plots

Almost every plot is clickable, which means that it will redirect you to different page (i.e. when you click on monster it can redirect you to its page) or filter existing records. In addition, if you hover on specific point/bar, it will display more information about specific record.

Tables

Every table is (on default) collapsed to minimize the amount of information website throws to you when you enter specific Page.
Every row is clickable, which means that it will redirect you to corresponding Page, i.e. for Monsters if you click on row in table, it will redirect you to specific Monster Build.
If row is not clickable, then there should be some icons or images, which are clickable. Check them!

I don't want my data to be there

I can assure you that no user will be able to tell if specific monster/rune/dungeon run is yours or not. However, if you want your data to be deleted, please contact me through Reddit (as for now the only form of contact) and we will see what can be done.

Wait, I know Windows App called Summoners War Statistics

The mentioned app has been made by me. It has been deprecated some time ago for few reasons, but - for me - the most important was that Windows App is... only for Windows users. Since first release, I've got a lot of messages that people can't use it becuase they aren't using Windows. I wanted everyone to be able to use this, so I decided to make an online version. If you were using previously mentioned app, you will notice that it is quite different than Windows App. Some features have been moved to online version, some developed, but few have been deleted. The biggest difference is, that it's Big Data instead of Per Person.
